Environment

What can I do?

User Action

Physical action the user performs, e.g. touching the screen, clicking

System Action

System response to the physical action, e.g. zoom level changes, device is locked

How do I do it?

Action Presentation

How affordance and system action are presented to the user, e.g. a scrollbar indicates scrollability

No Action Presentation

Availability of the action is not made explicit in the interface, e.g. keyboard shortcuts

Before

What is going to happen?

Preview

Gives information on the results of the action, before the action itself is started

Passive

Static elements previewing the outcome of the action, e.g. text on button

Active

User-triggered elements previewing the outcome of the action, e.g. Alt text

No Preview

No way to know what will happen without triggering the action, e.g. clicking icon with unclear meaning and no description

During

Is my input working?

Input Feedback

Confirms user input but gives no information on progress, e.g. active state of a button

No Input Feedback

No feedback on whether the input was received

What is happening?

Action Feedback

Continuous Feedback on current progress of the action

User-Controlled

User controls the progress of the action

Direct Manipulation

Interaction and result on the same object, e.g. drag & drop

Indirect Manipulation

Interaction and result on different objects, e.g. scrollbar

Autonomous

Action progresses autonomously, e.g. animation

No Action Feedback

No information on progress between initial and final state (hard cut)

After

What happened?

Response

Consequences of the action are clearly visible, e.g. the screen is locked after pressing power button

No Response

After the action it is not clear what exactly the effect was, e.g. copying to clipboard

How do I go back?

Reversibility

Effects of the action can be reversed easily

Inverse Action

Opposite of the action returns system to its previous state, e.g. moving i slider to its previous value

Unrelated Action

Different action enables returning to the previous state, e.g. back/undo button

No Reversibility

Not reversible without considerable effort or additional knowledge, e.g. recovering a file after deleting it
