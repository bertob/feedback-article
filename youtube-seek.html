Environment

What can i do?

User Action

Physical action the user performs

Discrete

Limited number of possible inputs, which are easily distinguishable, e.g. mouse click, tap screen

Continuous

Continuous spectrum of possible inputs, e.g. swipe screen, mouse movement, voice interfaces, 3D touch

System Action

System response to the physical action

Discrete

Limited number of possible outcomes, e.g. device is unlocked, app is opened

How do I do it?

Action Presentation

How affordance and system action are presented to the user, e.g. a scrollbar indicates scrollability

Before

What is going to happen?

Preview

Gives information on the results of the action, before the action itself is started

Passive

Static elements previewing the outcome of the action, e.g. text on button

Active

User-triggered elements previewing the outcome of the action, e.g. Alt text

During

Is my input working?

Input Feedback

Confirms user input but gives no information on progress, e.g. active state of button

What is happening?

No Action Feedback

No information on progress between initial and final state (hard cut)

After

What happened?

Response

Consequences of the action are clearly visible, e.g. the screen is locked after pressing power button

How do I go back?

Reversibility

Effects of the action can be reversed with little effort

Inverse Action

Opposite of the action will return system to its previous state, e.g. moving i slider to its previous value
